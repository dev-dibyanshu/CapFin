# Password Reset Testing Guide

## Overview
Forgot Password and Reset Password functionality has been implemented using ASP.NET Identity's built-in token generation.

## Features Implemented

### 1. Forgot Password API
- **Endpoint:** `POST /api/auth/forgot-password`
- **Security:** Prevents user enumeration (always returns success)
- **Token:** Uses ASP.NET Identity's `GeneratePasswordResetTokenAsync`
- **Expiry:** Tokens expire automatically (default: 1 day)
- **Output:** Logs reset link to console

### 2. Reset Password API
- **Endpoint:** `POST /api/auth/reset-password`
- **Validation:** Enforces password complexity rules
- **Token:** Single-use, automatically invalidated after use
- **Security:** Validates token using Identity's `ResetPasswordAsync`

## Testing Flow

### Step 1: Start AuthService

```bash
cd CapFinLoan.Backend/AuthService/CapFinLoan.Auth.API
dotnet run
```

Service will run on: `http://localhost:7001`

### Step 2: Request Password Reset

**Request:**
```http
POST http://localhost:7001/api/auth/forgot-password
Content-Type: application/json

{
  "email": "user@example.com"
}
```

**Response:**
```json
{
  "message": "If the email exists, a reset link has been sent."
}
```

**Console Output:**
```
========================================
PASSWORD RESET REQUEST
Email: user@example.com
Reset Link: http://localhost:5174/reset-password?email=user%40example.com&token=Q2ZESjhLV...
========================================
```

**Copy the token from the console output!**

### Step 3: Reset Password

**Request:**
```http
POST http://localhost:7001/api/auth/reset-password
Content-Type: application/json

{
  "email": "user@example.com",
  "token": "Q2ZESjhLV...",
  "newPassword": "NewPassword123!"
}
```

**Success Response:**
```json
{
  "message": "Password reset successful."
}
```

**Error Response (Invalid Token):**
```json
{
  "message": "Invalid or expired reset token."
}
```

### Step 4: Login with New Password

**Request:**
```http
POST http://localhost:7001/api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "NewPassword123!"
}
```

**Success Response:**

## Security Features

### 1. User Enumeration Prevention
- Forgot password always returns success, even if email doesn't exist
- Prevents attackers from discovering valid email addresses

### 2. Token Security
- Tokens are generated by ASP.NET Identity
- Automatically expire (default: 24 hours)
- Single-use only (invalidated after successful reset)
- Base64 URL-safe encoded for transmission

### 3. Password Validation
- Enforces ASP.NET Identity password policy
- Default requirements:
  - Minimum 6 characters
  - At least one uppercase letter
  - At least one lowercase letter
  - At least one digit
  - At least one special character

### 4. Error Handling
- Generic error messages to prevent information leakage
- Detailed logging for debugging (server-side only)

## Error Scenarios

```json
{
  "message": "Invalid or expired reset token."
}
```

### Weak Password
```json
{
  "message": "Invalid or expired reset token."
}
```
Note: Password validation errors are intentionally generic for security.

## Troubleshooting

### Token Decoding Error
- Ensure token is properly URL-encoded
- Copy entire token from console output
- Don't modify or truncate the token

### Token Expired
- Tokens expire after 24 hours by default
- Request a new reset link

### Password Doesn't Meet Requirements
- Check password complexity rules
- Ensure minimum 6 characters
- Include uppercase, lowercase, digit, and special character

### User Not Found
- Verify email address is correct
- Check if user exists in database
- Remember: API won't reveal if email exists (security feature)
